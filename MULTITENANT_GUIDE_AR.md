# 🏢 نظام Multi-Tenant - دليل استخدام Super Admin

## ✨ الميزات الجديدة

تم تطبيق نظام **Multi-Tenant** المتكامل يسمح لـ **Super Admin واحد** بـ:
- ✅ إنشاء حسابات جديدة (مثل Ahmed, Pierre, إلخ)
- ✅ إضافة مستخدم admin لكل حساب
- ✅ إدارة جميع الحسابات من لوحة تحكم واحدة
- ✅ كل حساب معزول تماماً عن الآخر

---

## 🚀 البدء السريع

### 1️⃣ إنشاء Super Admin الأول

**تشغيل السكريبت:**
```bash
python create_admin.py
```

**المخرجات:**
```
✓ تم إنشاء حساب Super Admin جديد
======================================================================
اسم المستخدم: <your-super-admin-username>
كلمة المرور: [يتم إدخالها عند التشغيل - لا يوجد كلمة مرور افتراضية]
البريد الإلكتروني: super@farm.local
الاسم: Super Administrator

💡 يمكن الآن استخدام هذا الحساب لـ:
   • إنشاء حسابات جديدة للشركات/المزارع
   • إضافة مستخدمين لكل حساب
   • إدارة جميع الحسابات من لوحة Super Admin
```

---

## 📊 واجهة المستخدم

### 🔐 تسجيل الدخول كـ Super Admin
1. ادخل **اسم المستخدم** الذي قمت بإنشائه في `create_admin.py`
2. ادخل **كلمة المرور** التي قمت بتحديدها عند إنشاء الحساب
3. اضغط **تسجيل الدخول**

### 📋 لوحة تحكم Super Admin

بعد تسجيل الدخول، ستجد في القائمة الجانبية قسم جديد:

```
===== Super Admin =====
🏢 إدارة الحسابات
```

اضغط على **"إدارة الحسابات"** لتصل لصفحة:
- قائمة جميع الحسابات المفعلة
- معلومات كل حساب (المسؤول، عدد المستخدمين، الحالة)
- خيارات التعديل والتحكم

---

## ➕ إنشاء حساب جديد

### الخطوات:

1. **اضغط على "حساب جديد"** في صفحة إدارة الحسابات
   
2. **ملأ بيانات الحساب:**
   - **اسم الحساب**: `Ahmed` أو `Pierre` أو أي اسم آخر
   
3. **ملأ بيانات مسؤول الحساب الأول:**
   - **اسم المستخدم**: `ahmed_admin` أو أي اسم
   - **البريد الإلكتروني**: `ahmed@farm.local`
   - **الاسم الكامل**: `Ahmed Mohamed`
   - **كلمة المرور**: `Password123`
   
4. **اضغط "إنشاء الحساب"** ✓

---

## 👥 إدارة مستخدمي الحساب

### من صفحة الحسابات:

1. **اضغط على أيقونة المستخدمين** 👥 بجانب الحساب
2. **اضغط "إضافة مستخدم"** لإضافة مستخدم جديد للحساب
3. **ملأ البيانات:**
   - اسم المستخدم
   - البريد الإلكتروني
   - الاسم الكامل
   - كلمة المرور

---

## 🔒 كيفية العزل

### كل حساب معزول تماماً:

✓ **مستخدم Ahmed** لا يرى بيانات Pierre
✓ **مستخدم Pierre** لا يرى بيانات Ahmed
✓ **كل حساب** له عماله الخاصة ومخزونه الخاص ومحاسبته الخاصة

### الحماية:
- البيانات تُفلتر تلقائياً حسب `account_id` من الجلسة
- أي محاولة للوصول لبيانات حساب آخر يتم حظرها
- كل عملية حفظ تتضمن التحقق من الحساب

---

## 🔑 أنواع المستخدمين

### 1️⃣ Super Admin
```
🔑 يمكنه:
   - إنشاء/تعديل/حذف الحسابات
   - إدارة جميع المستخدمين بكل الحسابات
   - الوصول لـ Dashboard إدارة الحسابات
```

### 2️⃣ Account Admin
```
🔑 يمكنه:
   - إدارة حسابه الخاص فقط
   - إضافة/تعديل/حذف مستخدمي حسابه
   - الوصول لجميع الأقسام (عمال، مخزون، محاسبة، إنتاج، إلخ)
   - فقط يرى بيانات حسابه الخاص
```

### 3️⃣ عضو عادي
```
🔑 يمكنه:
   - استخدام الأقسام التي أعطاها الـ Admin إذن الوصول لها
   - فقط يرى بيانات حسابه الخاص
```

## 📱 المسارات

```
🔓 عام (بدون تسجيل دخول):
   GET  /auth/login              → صفحة تسجيل الدخول
   POST /auth/login              → معالجة تسجيل الدخول
   POST /auth/logout             → تسجيل الخروج

🔐 Super Admin فقط:
   GET  /super-admin/accounts              → قائمة الحسابات
   GET  /super-admin/accounts/create       → صفحة إنشاء حساب
   POST /super-admin/accounts/create       → معالجة إنشاء حساب
   GET  /super-admin/accounts/<id>/edit    → تعديل حساب
   POST /super-admin/accounts/<id>/edit    → حفظ تعديلات الحساب
   POST /super-admin/accounts/<id>/toggle-status  → تفعيل/تعطيل
   GET  /super-admin/accounts/<id>/users           → مستخدمو الحساب
   GET  /super-admin/accounts/<id>/users/create    → إضافة مستخدم
   POST /super-admin/accounts/<id>/users/create    → حفظ المستخدم الجديد
```

## 🐛 استكشاف الأخطاء

### ❌ "ليس لديك صلاحيات كافية"
**السبب**: المستخدم الحالي ليس Super Admin
**الحل**: استخدم حساب يحتوي على `is_super_admin=True`

### ❌ "اسم الحساب موجود بالفعل"
**السبب**: اسم الحساب المدخل موجود فعلاً
**الحل**: استخدم اسماً مختلفاً

### ❌ "اسم المستخدم أو البريد الإلكتروني موجود بالفعل"
**السبب**: المستخدم موجود في النظام
**الحل**: استخدم بيانات مختلفة

### سيناريو 1: إنشاء حسابين لشركتين مختلفتين

```

📝 ينشئ حساب "أحمد العام"
    ├─ مع مسؤول: ahmed_admin | أحمد محمد

    └─ كلمة المرور: SecurePass123
    ↓
📝 ينشئ حساب "بيير العام"
    ├─ مع مسؤول: pierre_admin | Pierre Dubois
    ├─ البريل: pierre.admin@farm.com
    └─ كلمة المرور: SecurePass456
    ↓
✓ الآن كل شركة لها حسابها المستقل!
```

### سيناريو 2: إضافة موظفين للحساب

```
👤 ahmed_admin يسجل الدخول بحساب Ahmed
    ↓
📝 يذهب لـ إدارة النظام → إدارة الحسابات
    ↓
👥 يضيف موظفين جدد:
    ├─ supervisor_1 (مراقب)
    ├─ accountant_1 (محاسب)
    └─ warehouse_1 (أمين مخزن)
    ↓
✓ كل موظف يرى فقط بيانات حساب Ahmed
```
